A Brief Introduction to Data Structures and Algorithms

What Are Data Structures and Algorithms?

A data structure is a special way to store and organize data. There are many different data structures depending on the data involved and how one may want to manipulate it.

An algorithm is a set of instructions that solve a specific problem. You take a set of inputs, process it with your algorithm, and produce the desired output.

Why Study Data Structures and Algorithms?

Data structures and algorithms are essential to any computer science curriculum. Using the right data structure and algorithm can allow you to solve a wide variety of problems that software engineers face. Aside from simply solving a problem, you can use tools to analyze how fast an algorithm runs. This has a dramatic impact when it comes to solving problems that may involve enormous data sets or many users. A faster time complexity can be the difference between your program taking a few seconds versus several days!

What is Time Complexity?

Before going any further it's helpful to know about time complexity. It is a way to measure how fast an algorithm runs as the size of the input increases. We use big O notation to consider worst case time complexity. We use big O notation like O(n) or O(n^2). If an algorithm has time complexity O(n^2), then it means the number of steps it takes is proportional to n^2. Let's say an algorithm processes 10 numbers and has O(n) runtime. Then it could take 10 steps to process 10 numbers in the worst case. If the algorithm were slower and ran at O(n^2), then the algorithm could roughly take 10^2 steps to process the same amount of input. These numbers aren't precise but are intended to give an intution for time complexity. We can see that this becomes very important as n becomes large.
